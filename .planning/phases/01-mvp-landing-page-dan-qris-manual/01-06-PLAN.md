<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin browser → admin login | Credentials enter Laravel Auth/session boundary. |
| admin browser → verification routes | Privileged status changes enter order database. |
| admin browser → proof preview route | Admin views private proof images through authenticated controller. |
| buyer token URL → paid ZIP download | Public token link attempts access to private paid file. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-19 | Spoofing | admin login | mitigate | Laravel Auth::attempt, hashed passwords, session regeneration, throttle login, env-only seeded credentials. |
| T-01-20 | Elevation of Privilege | admin routes | mitigate | auth middleware plus is_admin check on dashboard, verification, and proof preview routes; block non-admin users. |
| T-01-21 | Tampering | confirm/reject actions | mitigate | CSRF-protected POST routes, admin-only authorization, validated reject_reason, exact state transitions. |
| T-01-22 | Information Disclosure | paid ZIP download | mitigate | Verify order status before Storage::disk('local')->download; 403 for unverified states; no public URL. |
| T-01-23 | Information Disclosure | proof preview | mitigate | Proof preview streams from local private disk behind auth + admin check; no public proof URLs exposed. |
| T-01-24 | Repudiation | manual verification | mitigate | Store verified_at and verified_by, and keep payment_proofs status history. |
</threat_model>
