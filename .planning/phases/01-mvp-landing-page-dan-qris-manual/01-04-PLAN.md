<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| visitor browser → lead/order forms | Untrusted form input enters Laravel controllers. |
| public lead token URL → private lead file | Token authorizes lead magnet page/download without login. |
| Blade output → browser | User-provided email/validation messages render back into HTML (escaped by Blade {{ }}). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-09 | Spoofing | lead download_token | mitigate | Use high-entropy unique token; invalid token returns 404. |
| T-01-10 | Information Disclosure | lead magnet download | mitigate | Stream from local private disk only after token lookup; no public storage URL. |
| T-01-11 | Tampering | lead capture form | mitigate | StoreLeadRequest validates email only; CSRF web middleware and throttle route from Plan 01-02. |
| T-01-12 | Repudiation | duplicate lead opt-ins | mitigate | Keep first_opted_at and update last_opted_at for every duplicate submission. |
| T-01-13 | Information Disclosure | landing page copy/assets | mitigate | No fake testimonials/logos/customer claims; only approved trust badges and product snippets. |
</threat_model>
