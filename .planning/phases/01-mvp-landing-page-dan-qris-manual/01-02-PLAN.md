<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| visitor browser → Laravel web routes | Untrusted forms for order and lead capture cross into server-side validation (defined as Form Request contracts here, implemented in downstream plans). |
| buyer browser → proof upload | Untrusted image file enters private storage (contract defined here). |
| admin browser → admin routes | Credential-protected actions change payment state (routes defined here). |
| public token URL → order/lead lookup | Tokenized links grant status/download affordances without buyer login. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | admin seed/login foundation | mitigate | AdminUserSeeder consumes env-only credentials; route contract uses auth middleware on admin group. |
| T-01-02 | Tampering | migrations/status contracts | mitigate | Restrict valid order statuses to pending/proof_submitted/verified/rejected via model logic in downstream plans. |
| T-01-03 | Information Disclosure | private file paths | mitigate | Route contract uses {invoice_token}/{download_token} for buyer URLs, never numeric IDs. |
| T-01-04 | Tampering | proof upload validation contract | mitigate | Form Request enforces required file image mimes jpg,jpeg,png,webp max 4096. |
| T-01-05 | Information Disclosure | invoice_token/download_token schema | mitigate | Schema uses unique high-entropy string tokens as primary public identifiers. |
</threat_model>
