<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| visitor browser → order creation | Untrusted buyer identity/contact fields enter database. |
| buyer browser → proof upload | Untrusted image file enters private storage and metadata table. |
| public invoice token URL → order record | Token grants access to order status and upload form. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-14 | Spoofing | invoice_token | mitigate | Generate high-entropy unique token; use token lookup only; never expose numeric order IDs. |
| T-01-15 | Tampering | proof upload | mitigate | Form Request image/mime/size validation, private local disk, generated file names, no trusted client extension. |
| T-01-16 | Information Disclosure | payment proofs | mitigate | Store proofs on local private disk and never link public storage paths. |
| T-01-17 | Denial of Service | order/proof routes | mitigate | Use route throttle contracts from Plan 01-02 and max 4096 KB upload limit. |
| T-01-18 | Repudiation | proof history | mitigate | Create payment_proofs history rows with timestamps and status rather than overwriting metadata. |
</threat_model>
